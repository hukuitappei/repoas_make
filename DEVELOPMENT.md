## 共通ルール

### コーディング規約
```
クラス名・メソッド名  : PascalCase
プライベートフィールド : _camelCase
公開プロパティ        : PascalCase
定数                  : UPPER_SNAKE_CASE
```
- MonoBehaviour は薄く保つ（ロジックは純粋 C# クラスへ）
- ScriptableObject でマスターデータを管理
- マジックナンバー禁止（定数・ScriptableObject に切り出す）

### フォルダ構成
```
Assets/
├── Scripts/
│   ├── Core/       ← コアシステム（Claude Code）
│   ├── Buildings/  ← 施設クラス（Codex）
│   ├── Guilds/     ← ギルドクラス（Codex）
│   ├── UI/         ← UI（Codex）
│   └── Data/       ← ScriptableObject 定義
├── Sprites/        ← [カテゴリ]_[名前]_[variant]
├── Audio/
├── Prefabs/
└── ScriptableObjects/
```

### 素材命名規則
```
facility_farm_lv1.png
guild_warrior_member.png
ui_button_confirm.png
audio_bgm_main.wav
```

### Git 運用
- ブランチ：`main`（安定）/ `develop`（開発）/ `feature/xxx`（機能）
- コミット粒度：1機能・1修正につき1コミット
- バイナリ素材は Git LFS で管理
- 節目でタグ：`v0.1`（コアループ）`v0.2`（UI完成）`v0.3`（バランス調整済み）

### Claude Code / Codex 連携プロトコル
1. 🔵 Claude Code が **仕様書（Markdown）** を作成
2. 👤 ユーザーが Codex に仕様書を渡して実装依頼
3. 🟠 Codex が実装・コード提出
4. 🔵 Claude Code がコードレビュー・修正指示
5. 問題なければ `develop` にマージ
